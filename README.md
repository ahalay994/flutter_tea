# Tea App

Приложение для управления и каталогизации различных сортов чая. Предоставляет функциональность для просмотра, добавления и организации информации о чае, включая страну происхождения, тип, внешний вид, вкусы, инструкции по завариванию и изображения.

## Основные возможности

- **Просмотр каталога чаёв** с возможностью детального просмотра
- **Добавление новых записей** о чаёв с полным описанием
- **Организация информации** о чаёв: страна, тип, внешний вид, вкусы
- **Инструкции по завариванию** с форматированием текста
- **Загрузка и отображение изображений**
- **Фильтрация и поиск** по различным критериям
- **Оффлайн режим** - возможность использования приложения без интернет-соединения
- **Автоматическая синхронизация** данных при подключении к интернету

## Функциональность фильтрации

- **Мультиселект фильтры** по странам, типам, внешнему виду и вкусам
- **Текстовый поиск** по названию, описанию, инструкциям по завариванию, температуре и весу
- **Работа в оффлайн режиме** - фильтрация работает даже без интернет-соединения
- **Индикаторы активных фильтров** в интерфейсе
- **Кнопки "Применить" и "Сбросить"** для управления фильтрами

## Архитектура

- **Flutter**: Кроссплатформенный фреймворк для мобильных приложений
- **Dart**: Язык программирования для Flutter
- **Riverpod**: Решение для управления состоянием
- **Supabase**: Backend-as-a-Service для хранения данных
- **SQLite**: Локальное хранилище для оффлайн режима
- **HTTP**: Для взаимодействия с API

## Технологии

- **Flutter**: Кроссплатформенный фреймворк для разработки мобильных приложений
- **Dart**: Язык программирования
- **Riverpod**: Управление состоянием
- **SQLite**: Локальное хранилище данных

- **Image Picker**: Выбор и загрузка изображений

## Установка и запуск

1. Клонируйте репозиторий
2. Установите зависимости: `flutter pub get`
3. Создайте файл `.env` в корне проекта с содержимым:
   ```
   API_URL=your_api_base_url
   APP_NAME=Tea App
   ```
4. Запустите приложение: `flutter run`

## Структура проекта

- `lib/`: Основной исходный код приложения
- `lib/api/`: Клиенты API и DTO
- `lib/controllers/`: Контроллеры бизнес-логики
- `lib/models/`: Модели данных
- `lib/screens/`: Экраны приложения (домашний, добавление, детали)
- `lib/providers/`: Провайдеры Riverpod
- `lib/widgets/`: Повторно используемые компоненты UI
- `assets/images/`: Статические изображения

## Оффлайн режим

Приложение полностью функционально в оффлайн режиме:
- Данные сохраняются в локальной базе данных SQLite
- Фильтрация и поиск работают на локальных данных
- При восстановлении подключения происходит автоматическая синхронизация
- Индикатор статуса подключения в интерфейсе
